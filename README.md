# pages-youzhibo-guide

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库中的页面均为静态 HTML 文件，可独立运行，不依赖特定后端服务。本仓库本身不针对任何单个域名或具体网站，仅作为页面资源的存储与分发通道。

## 目录说明

```
.
├── README.md          # 本文件
├── index.html         # 仓库首页 / 页面导航
├── pages/             # 主要页面存放目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/            # 共用资源（CSS、JS、图片等）
    ├── style.css
    ├── script.js
    └── images/
```

- `index.html`：仓库的入口页面，包含页面索引或简单导航。
- `pages/`：存放所有独立 HTML 页面文件，每个文件为完整可访问的页面。
- `assets/`：存放多个页面共用的样式、脚本、图片等资源，避免重复。

## 页面归档说明

- 所有页面均为纯前端 HTML 文件，不包含服务器端脚本。
- 页面可直接在浏览器中通过文件路径或 GitHub Pages 等静态托管服务访问。
- 页面之间无强制依赖关系，可单独使用。
- 归档内容以通用性为主，不针对特定网站或服务。

## 维护说明

- 欢迎通过 Pull Request 提交新的页面或改进现有页面。
- 提交页面时请确保：
  - 页面为独立 HTML 文件，路径放置在 `pages/` 目录下。
  - 若有共用资源，请放入 `assets/` 对应子目录，并在页面中正确引用相对路径。
  - 不包含指向具体商业网站、营销内容或诱导点击的链接。
- 本仓库不保证对所有页面的长期可用性，请自行备份重要内容。
- 如有疑问或建议，请提交 Issue 讨论。
